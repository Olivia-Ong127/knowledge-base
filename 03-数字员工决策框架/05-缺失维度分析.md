# 框架缺失维度分析

> 最后更新：2026年6月16日

---

当前框架（v0.5）覆盖了"决策流程"——但它离一个完整的"数字员工"还有差距。以下是在对话中识别出的缺失维度。

## 缺失一：目标体系 🔴核心缺失

**问题：** 调拨决策是在优化什么？

当前框架讲了"怎么做"，但没讲"为了什么而做"。

| 可能的目标 | 相互冲突吗？ |
|-----------|------------|
| 最大化售罄率 | 可能与毛利率冲突 |
| 最小化库存积压 | 可能与售罄率冲突 |
| 平衡毛利率与周转 | 需要权衡 |
| 不同场景（季初vs季末）目标不同 | 需要区分 |

**影响：** 没有目标，决策就没有方向。同一个调拨建议，目标不同，结论可能完全相反。

**待处理：** 访谈联创A时需要确认——他最关心的指标是什么。

## 缺失二：技术路径选择 🔴核心缺失

**问题：** 这个数字员工到底用什么技术做出来？

当前框架只写了一个粗略的阶段排期表（Phase 1/2/3），但缺少真正的技术选型讨论。

**三道选择题需要回答：**

| 选择题 | 选项 | 当前建议 |
|-------|------|---------|
| 规则驱动 vs AI驱动？ | A=规则 B=AI C=先规则后AI | C |
| 看今天 vs 看趋势？ | A=当下 B=趋势 C=先当下后趋势 | C |
| 等数据 vs 先跑仿真？ | A=等数据 B=仿真 C=双线并行 | C |

> 详见 `04-技术路径选择/01-三道选择题.md`

## 缺失三：分级决策/自治度 🟡重要缺失

**问题：** AI能自己做什么？什么必须人批？

当前框架只说"AI建议，人决策"，太笼统。

| 级别 | 含义 | 当前框架有吗？ |
|------|------|--------------|
| L1-信息提醒 | "某SKU存销比异常" | 🟡 部分 |
| L2-建议参考 | "建议调5件，预计收益XXX" | ✅ 有 |
| L3-自动执行 | 低风险场景AI直接执行，人兜底 | ❌ 没有 |
| L4-人工干预 | 高风险必须人批 | ❌ 没有分级标准 |

## 缺失四：异常兜底机制 🟡重要缺失

**问题：** 出错了怎么办？

当前框架假设的是数据完整、逻辑通畅的理想路径。现实场景：

| 异常情况 | 有预案吗？ |
|---------|----------|
| 数据缺失/延迟 | ❌ 没有 |
| 多个规则冲突（地理优先vs库存优先） | ❌ 没有 |
| AI置信度低（不确定时） | ❌ 没有 |
| 人否决了AI建议 | ❌ 没有记录/学习机制 |

## 缺失五：品类差异化 🟢可后补

**问题：** 不同品类的调拨逻辑可能完全不同。

| 品类 | 调拨逻辑差异 |
|-----|------------|
| 基本款（白T恤） | 全年可调，关注库存深度 |
| 潮流款 | 窗口期短，速度优先 |
| 配饰 | 可能不调拨直接退仓 |
| 季末折扣款 | 目标不是卖高价而是清库存 |

框架目前是一套规则打天下，不同品类需要差异化处理。
