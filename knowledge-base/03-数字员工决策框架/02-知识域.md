# 知识域：它需要知道什么

> 最后更新：2026年6月16日

---

数字员工做调拨决策，需要掌握五类知识：

## 一、商品知识

- **品类层级：** 大类（服装/鞋品/配饰）→ 中类（上衣/裤/裙/外套）→ SKU（款号+颜色+尺码）
- **生命周期：** 新品导入期（30-45天）→ 快速成长期 → 成熟稳定期 → 衰退出清期 → 退市清仓期
- **商品属性：** 季节、波段、价格带、风格、材质、连带关系
- ⚠️ 需验证：实际品类层级深度、生命周期判断标准

## 二、库存知识

- **多维度视图：** 总部仓、区域仓、门店、在途、退货、预售
- **健康度指标：** 存销比（核心）、周转率、库龄、缺货率
- **状态标签：** 可售、预留、不可动、残次、冻结
- ⚠️ 需验证：存销比健康值范围、库龄预警阈值

## 三、调拨规则知识（核心）

**8种触发条件：**

| 触发类型 | 频率 | 说明 |
|---------|------|------|
| 缺货调拨 | 高 | A店缺货，从有货的店调 |
| 滞销调出 | 中 | 某款在某店走不动，调去好卖的店 |
| 计划铺货 | 周期高频 | 新品上市按计划铺到各店 |
| 促销备货 | 活动低频 | 大促前集中备货 |
| 季节性收仓 | 季节低频 | 换季时收尾货回仓 |
| 应急调拨 | 偶发 | 突发缺货/异常 |
| 优化调拨 | 低频 | 主动优化库存分布 |
| 逆向调拨 | 中低频 | 从门店退回仓库 |

**供给匹配四层过滤：**
1. 地理约束——同区域优先
2. 库存充裕度——按存销比排序
3. 商品状态匹配——同款优先
4. 业务红线——VIP店不可调、安全库存不可动

## 四、业务场景知识

7种典型场景：正常补货、新品铺货、促销活动、季末出清、爆款追单、突发缺货、滞销清理

## 五、外部参考知识

历史调拨记录、销售趋势、天气、节假日、区域消费力
